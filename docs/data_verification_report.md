## 주요 발견 사항

### 1. 공공데이터포털 신규 명세 (resultCode 3자리)
설계서는 `resultCode == "00"` 기준이지만, 실제 응답은 `"000"`. 검증 스크립트에서 둘 다 허용하도록 수정. 본격 수집 코드에도 동일 적용 필요.

### 2. 연립다세대 응답에 대지권면적 없음
설계서가 가정한 `대지권면적` 필드가 실제 응답에 없음. 사용 가능 필드: `buildYear, dealDay/Month/Year, deposit, excluUseAr, floor, houseType, jibun, mhouseNm, monthlyRent, sggCd, umdNm` 등. 모델 피처 재정의 필요.

### 3. KOSIS 청년 연령 구간 단위
`DT_118N_LCE0004`는 5세 단위 연령(29세이하·30~39세·40~49세·...)만 제공. 청년(19~34세) 정밀 추출 불가. 두 구간 가중평균으로 근사하거나, 통계청에서 1세 단위 임금 통계표를 별도로 찾아야 함.

### 4. 행안부 인구 + R-ONE + 통계누리 — 엔드포인트 미확정
3건 모두 키 자체는 발급됐지만 정확한 OpenAPI 진입점 / 필수 파라미터 명세를 알아야 호출 가능. 추측 엔드포인트로는 모두 실패. 사용자 확인 후 재검증 필요.

## 사용자 액션 필요 항목 (3건)

### A. 행안부 주민등록인구 OpenAPI URL
- 위치: 공공데이터포털(data.go.kr) → 마이페이지 → 활용신청 현황 → "행정안전부_통계연보_지역별 주민등록인구" 상세 페이지
- 확인 필요: `End Point` URL, 필수 요청 변수명, 응답 형식 (JSON / XML / CSV)

### B. R-ONE OpenAPI 베이스 URL + 필수 파라미터
- 위치: R-ONE(reb.or.kr/r-one) → OpenAPI 가이드 또는 마이페이지
- 확인 필요: 필수 `SERVICE` 값 (예: `SttsApiTblData` 등), 필수 `STATBL_ID` 형식

### C. 통계누리 OpenAPI 진입점
- 위치: 국토교통 통계누리(stat.molit.go.kr) 로그인 후 OpenAPI 메뉴
- 확인 필요: 통계표 목록 조회 URL, 데이터 조회 URL, 인증 헤더/쿼리 파라미터 형식
